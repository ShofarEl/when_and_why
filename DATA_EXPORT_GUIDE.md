# Data Export Guide

This guide explains how to export participant data from the MongoDB database to CSV format for analysis.

## Quick Start

### Option 1: Using npm script (Recommended)
```bash
npm run export
```

### Option 2: Using batch file (Windows)
```bash
export-data.bat
```

### Option 3: Using shell script (Mac/Linux)
```bash
chmod +x export-data.sh
./export-data.sh
```

### Option 4: Direct execution
```bash
cd server
node scripts/exportToCSV.js
```

## What Gets Exported

The script creates 7 CSV files in the `server/exports/` directory:

### 1. `participants_YYYY-MM-DD.csv`
Main participant information:
- Participant ID
- Demographics (age, academic level, major)
- Data science familiarity
- AI experience
- Prior courses
- Condition order

### 2. `sessions_YYYY-MM-DD.csv`
Experimental session data:
- Session details (timing, reflection condition)
- Task completion info
- Ideas count
- AI suggestions count
- Interactions count
- Questionnaire responses (agency, dependence, cognitive load)

### 3. `ideas_YYYY-MM-DD.csv`
All ideas generated by participants:
- Idea content
- Timestamp
- Session context
- AI influence indicators

### 4. `interactions_YYYY-MM-DD.csv`
All user interactions:
- Action type (clicks, AI requests, etc.)
- Timestamp
- Session context
- Interaction details (JSON)

### 5. `transfer_tasks_YYYY-MM-DD.csv`
Transfer task performance:
- Task number
- Ideas generated
- Completion time
- Timestamp

### 6. `post_study_YYYY-MM-DD.csv`
Post-study survey responses:
- Condition preference ratings
- Learning rating
- AI usefulness rating
- Open-ended feedback

### 7. `ai_suggestions_YYYY-MM-DD.csv`
AI suggestions provided:
- Suggestion content
- Timestamp
- Acceptance/dismissal status
- Session context

## File Location

All exported files are saved to:
```
server/exports/
```

Files are named with the current date to prevent overwriting previous exports.

## Requirements

- Node.js installed
- MongoDB connection configured in `server/.env`
- `MONGODB_URI` environment variable set

## Troubleshooting

### "Cannot connect to MongoDB"
- Check your `server/.env` file
- Verify `MONGODB_URI` is correct
- Ensure MongoDB Atlas allows your IP address

### "No data to export"
- Verify participants have completed the study
- Check MongoDB database has data

### "Module not found"
```bash
cd server
npm install
```

## Data Analysis Tips

### Loading in R
```r
library(readr)

participants <- read_csv("server/exports/participants_2024-01-15.csv")
sessions <- read_csv("server/exports/sessions_2024-01-15.csv")
ideas <- read_csv("server/exports/ideas_2024-01-15.csv")
```

### Loading in Python
```python
import pandas as pd

participants = pd.read_csv('server/exports/participants_2024-01-15.csv')
sessions = pd.read_csv('server/exports/sessions_2024-01-15.csv')
ideas = pd.read_csv('server/exports/ideas_2024-01-15.csv')
```

### Loading in Excel
1. Open Excel
2. Data → From Text/CSV
3. Select the CSV file
4. Follow the import wizard

## Privacy & Ethics

- All data is anonymized with participant IDs
- No personally identifiable information is exported
- Store exported files securely
- Follow your institution's data protection guidelines

## Automated Exports

To schedule regular exports, you can use:

### Windows Task Scheduler
1. Open Task Scheduler
2. Create Basic Task
3. Set trigger (e.g., daily)
4. Action: Start a program
5. Program: `export-data.bat`

### Mac/Linux Cron
```bash
# Edit crontab
crontab -e

# Add daily export at 2 AM
0 2 * * * /path/to/export-data.sh
```
